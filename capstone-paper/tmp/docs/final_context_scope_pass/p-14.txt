Scope


The proposed system provides module-based access for seven core roles: super admin,
admin, registrar, finance, teacher, student, and parent. Access is structured by institutional
responsibility and each module groups related actions into one operational unit.

The scope centralizes enrollment, finance, academic, governance, and stakeholder
communication workflows while assigning each role to complete modules that include
transaction handling, validation, and analytics interpretation.

The Registrar can access the following:

    ● Enrollment Lifecycle Module - Manage intake creation, profile updates, and
        payment-linked progression within one registrar workflow.
    ● Learner Records and Directory Module - Maintain searchable learner profiles,
        identity continuity, and authorized record retrieval.
    ● SF1 Reconciliation Module - Upload SF1 records and execute LRN-based
        enrichment aligned with LIS-oriented validation rules.
